# Backlog de ajustes UX post 10D

Este backlog registra observaciones pendientes que no forman parte de los Bloques 10D-3 ni 10D-4. No deben mezclarse con terminologia, microcopy, trazabilidad visual ni con cambios de reglas academicas.

## Navegación

- Revisar ajustes residuales de jerarquía en sidebar cuando existan más módulos operativos.
- Validar con usuarios institucionales si las agrupaciones actuales de reportes requieren accesos favoritos por perfil.
- Revisar rutas de soporte técnico y enlaces heredados que puedan quedar como respaldo administrativo.

## Formularios

- Refinar formularios densos de administración, catálogos, trayectoria, movimientos y periodos para mejorar orden visual.
- Evaluar búsquedas relacionales más profundas para discentes, grupos, programas y periodos cuando el backend ofrezca endpoints específicos.
- Separar flujos futuros de restablecimiento de contraseña de la edición ordinaria de usuarios.

## Filtros

- Sustituir filtros que siguen siendo texto libre por selectores seguros cuando exista fuente backend autorizada.
- Agregar cascadas completas periodo -> carrera -> grupo -> asignatura en reportes, si se aprueba como mejora posterior.
- Revisar filtros nominales para evitar captura de datos sensibles o ambiguos.

## Terminología y microcopy

- Mantener revision editorial posterior para textos largos que requieran validacion institucional.
- Validar con usuarios si algun documento oficial debe conservar "Unidad de aprendizaje" en lugar de "Asignatura".
- Revisar microcopy de confirmaciones criticas durante QA de 10E, sin alterar reglas academicas.

## Ajustes visuales menores

- Evaluar migracion futura a una libreria comun de iconos si el proyecto decide agregar dependencia.
- Normalizar densidad de tablas largas en reportes e historial interno.
- Afinar comportamiento responsive de filtros y tablas densas durante QA integral 10E.

## Decisiones funcionales pendientes

- Definir si jefatura de carrera requiere acceso granular a eventos críticos por ámbito de carrera.
- Validar si se necesita una consulta pedagógica específica más allá de reportes existentes.
- Definir alcance futuro de asignación docente en portal; la apertura de periodo no asigna docentes automáticamente.

## Referencia posterior a 10E

QA 10E consolida los pendientes no bloqueantes en `docs/qa_10e/backlog_post_10e.md`. Este backlog 10D se conserva como antecedente historico y no debe usarse para reabrir mejoras dentro del cierre MVP salvo que se reclasifiquen como P0/P1.
